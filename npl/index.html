National Physical Laboratory (CSIR-NPL)

Monitoring Dashboard for Cesium Fountain Atomic Clock (CsF1)

A unified digital tool for precise timekeeping and critical parameters monitoring

Overview

Developed during a research internship at the CSIR-National Physical Laboratory, this project replaces a fragmented legacy system with a centralized, real-time monitoring platform. The dashboard tracks critical environmental and instrumental parameters for the CsF1 (Cesium Fountain - 1), the system responsible for maintaining Indian Standard Time (IST).

The Stakes: The CsF1 defines the SI second. Microscopic fluctuations in ambient conditions or laser alignment can introduce measurement errors.

Context &
Challenges

Operating the CsF1 previously required managing isolated and fragile monitoring tools. Data was scattered, and manual oversight risked the stability of national time standards.

Dispersed Data

Essential metrics (Lasers, photodiodes, temp & humidity) were scattered across different hardware controllers.

Manual Monitoring

Scientists had to physically check instruments or log data manually, prone to human error.

Latency

Critical fluctuations in laser alignment needed to be spotted instantly, which legacy systems couldn't do.

Constraints &
Users

The "Dark Room" Constraint

The CsF1 operates in strict laboratory conditions. A primary design constraint was the lighting: the interface could not emit high-luminance white light, as stray photons could interfere with the atomic transition experiment. A custom neon-dark theme was engineered to prevent light pollution.

Designing for Domain Experts

  • Head Scientists: Require immediate oversight of system uptime, operational health, and connection status.
  • Senior Scientists: Rely on live, auto-refreshing graphs to observe the immediate effects of physical interventions (e.g., tweaking laser alignment).
  • PhD Students: Need robust historical analysis tools to download CSV logs for root-cause analysis after overnight failures.

System
Architecture

The solution required a ground-up rebuild of the data acquisition pipeline, ensuring isolated hardware could communicate with a single Python backend.

Data Acquisition & Logging

A FLUKE 1620A Thermo-Hygrometer tracks temp/humidity via serial ports, while an ESP8266 microcontroller digitizes 13 channels of laser and photodiode signals, transmitting them wirelessly to a Flask API. Every reading is tagged with localized IST, UTC, and Modified Julian Date (MJD) timestamps to ensure scientific consistency.

Interface
Breakdown

Live Sensor Cards & Navigation

Built using Plotly Dash, the interface translates raw CSV logs into an interactive dashboard. A collapsible sidebar provides dedicated views, while vital parameters display numerical values and color-coded status indicators that refresh every 10 seconds.
