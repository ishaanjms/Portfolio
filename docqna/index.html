Designing an assistant that makes AI research analysis inspectable midway.

DocQnA already worked with large transcript collections. The new assistant feature explored a more direct interaction: a user enters one natural-language prompt and receives quantitative evidence generated from many interviews. The core UX challenge was not only generation, but how to reveal generated data at the right point in the pipeline.

01

UX Problem

In research tools, waiting until the end of an AI pipeline to show data creates distrust. Users need enough visibility to understand what the system is extracting, where it is uncertain, and how the output is forming.

Visibility gap

Generated data appeared too late.

Users could not easily inspect intermediate quantitative patterns before committing to the final synthesis.

Control gap

One prompt still needed steering.

The assistant had to support broad prompts while making scope, source coverage, and generated measures understandable.

Trust gap

Numbers needed traceability.

Quantitative outputs from qualitative transcripts need confidence signals, source references, and clear states.

03

Design Decisions

Show generated data before final synthesis.

The preview stage gives users a chance to inspect counts, themes, and confidence before the assistant turns them into a polished answer.

Treat AI output as editable research material.

The UX avoids presenting AI-generated numbers as final truth. Instead, it frames them as working evidence that can be refined.

Keep source scale visible.

Users need to know whether an output came from all transcripts, a filtered subset, or a partial extraction state.

Outcome

A clearer AI workflow for transforming qualitative interviews into quantitative evidence.

Better pipeline trust

Users could see how generated data was forming instead of waiting for an opaque final answer.

More usable quant output

The assistant framed generated tables and counts as inspectable, refinable research artifacts.
